a talk in three acts

Less is more.

In this talk, we will look at three interesting techniques: one explores how we can reduce the memory footprint of vectors, another shines light on how geometric constraints can enable training with less data and the last one rethinks the architecture of large models to get more out of the same compute budget.
Open here. The whole talk is one argument: the next leap may not be more scale, but noticing where representations are wasteful, and that waste has exploitable geometric structure. Three papers, three layers (the rail at bottom-left tracks which one we are on). Every demo is live. Drag them.

part 01 · storage

The KV cache eats your memory.

The first cost is memory. To write each token, attention reads every earlier one, so the model caches a key and a value per token, per layer, per head. That pile grows with the context.
Served at scale, memory requirement for KV cache can pass the weights. However, the attention mechanism only cares about inner product between key and query vectors.
KV CACHE vs MODEL WEIGHTS
Frame the cost. The cache is linear in context and at long context it passes the weights. INT4 is the obvious 4x win. Tee up why it fails: outliers. Slide the context to show the crossover.

part 01 · storage
ONE GRID · 16 levels for every coordinate

A fixed grid is only as good as its widest coordinate.

Real key and query vectors are lopsided. Whatever budget of levels you pick, they all have to stretch up to that one outlier, so the small coordinates collapse onto the bottom and round to almost nothing. More bits cannot fix a bad shape.
levels
small coords lost
0 / 15
levels actually used
16 / 16
So the fix is not more bits. It is to change the vector's shape before rounding.
The bridge from quantizing to rotating. One fixed grid serves the whole vector, so its level spacing is set by the largest coordinate. On a lopsided real key, the outlier stretches the grid and the small coordinates round to the same level, indistinguishable. Grow the outlier and watch the small coordinates get lost and the used levels collapse toward two. The point: you cannot bit your way out of a bad shape, so reshape the vector first. The tool is a rotation, which is the next slide.

RECONSTRUCTION ỹ vs y · the tails get capped

But it lies about inner products.

Rounding sends each coordinate to its bin's average, which sits closer to zero, so the key comes back shrunk. That is fine for reconstruction, but an inner product with a shrunk key comes out too small, the same way every time, and averaging cannot fix a bias.

At one bit the score keeps only 2/π, about 0.64 of its value, on every key. So reconstruction solved one problem, and the inner product is a different one.
The bias, the heart of the paper. The MSE-optimal decode is the bin mean, which sits closer to 0 than the inputs, so the reconstruction is a shrunken copy. The staircase shows ỹ(y) sitting inside the identity line: shrinkage. The inner-product multiplier is λ_b = E[ỹ²]/σ² (0.64 / 0.88 / 0.97 for b=1,2,3). Same factor every trial, so averaging does not help.

INNER-PRODUCT ESTIMATE · truth = 1.00

One bit makes it honest.

The leftover gap is the residual. Keep only its sign, which is one bit, and since we know the average size of such a gap in advance, the sign says which way to nudge and the size says how far. Add it back and the bias cancels. Any single estimate is noisier, but the average is exactly right.

Same one bit, the difference is where it is spent. The biased mean parks at 0.64; the corrected one reaches the truth.
QJL, the un-biaser. Fire trials: the round-only estimate clusters tightly but its mean sits at 0.64 (biased, never moves). The +residual-bit estimate scatters wider but its running mean converges to 1.00 (unbiased, higher variance). Bias vs variance, made visible. This is TurboQuant-prod.

DISTORTION vs bits · log scale

How close to the best possible?

Information theory sets a floor no quantizer can beat: error cannot fall below 4 to the minus b. TurboQuant lands about 2.7x above it, 1.5x at one bit. And its error falls like 4 to the minus b, exponential in bits, where older sketches only improved like 1/b.

part 02 · learning
HIDDEN STATE · one point per token

A sentence is a path.

Read a token and the hidden state steps to a new point; read the next and it steps again. String those points together and the sentence becomes a path through space, like a walk through a city.
The foundational picture for the act. Tokens arrive, the hidden state walks point to point, tracing a path. Replay it once. After "sentence equals path", the geodesic and the tube need no extra setup.

TWO STEPS · the angle between them

Two steps, one angle.

Walk the path two steps at a time, this step and then the next. If you are walking a straight line, the two point the same way, so the angle between them tells the whole story: it is zero on a straight stretch, and it opens up as the path bends.

part 02 · learning

It is one extra term in the loss.

STP comes from JEPA, Yann LeCun's idea of predicting in representation space instead of in tokens. The tube is that idea for language, and it needs none of JEPA's usual machinery.
the whole objective
L = LNTP  +  λ · LSTP  ( 1 − cos of the bend )
what jepa usually needs
Two augmented views of the input, a separate predictor network, and an extra forward pass.
what the tube needs
One cosine on the model's own trajectory. No second view, no predictor. And λ is zero at inference, so it costs nothing to run.
The method, and the JEPA framing the paper is named for. STP is a JEPA-style regularizer for language: predict in representation space, but using the sequence's own trajectory instead of two augmented views. The full objective is just L_NTP + λ·L_STP with λ ≈ 0.01 to 0.08. No predictor network (identity is optimal under the geodesic hypothesis), no extra forward pass, cheaper than the group's earlier LLM-JEPA. And λ = 0 at inference, so zero runtime cost. One term, no new machinery.

INFERENCE CONE · two meanings

Wobble kills diversity.

Two close meanings are two paths, and clean paths behave like smooth flows that never cross. But generation feeds on its own output, so noise compounds, fanning each path into a widening cone. Turn STP off, the cones overlap and the two meanings collapse into one. Turn it on, the wobble shrinks and they stay distinct.

The diversity story. Toggle STP off: the two ensembles bleed together and merge to one mode collapse. On: the cones tighten and stay separated. This is why straightness preserves diversity.

part 03 · compute

Two ways to spend less compute.

conditional compute · moe
For each token, route to a few experts out of many, so the model picks which weights to run. This part is already standard.
conditional memory · engram
For each token, fetch a few rows from a huge table, so the model picks which memories to recall, with no matrix multiply, just a read.
A fact you can look up should not cost a full expert. Engram puts the second kind of sparsity next to the first.
Frame the new axis. MoE chooses which computation to run. Engram chooses which memory to read. They are complements. The next slides show how Engram does the read and how to balance the two.
